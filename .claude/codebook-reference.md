# Obsidian + Claude Code Codebook — 12 Commands Reference

Source: "Obsidian + Claude Codebook (doc).docx" by Vin (Internet Vin)

## Commands

1. **/context** — Load full life/work state: projects, preferences, priorities, current focus
2. **/today** — Morning planning: calendar, tasks, daily notes → prioritized plan
3. **/trace** — Track idea evolution over time across vault, backlinks, timeline
4. **/connect** — Bridge two domains using link graph, find unexpected connections
5. **/ghost** — Answer a question as the user would, using vault writing/beliefs
6. **/challenge** — Pressure-test beliefs, find contradictions and weak points
7. **/ideas** — Vault scan → idea report: tools to build, people to meet, topics, things to write
8. **/graduate** — Extract undeveloped ideas from daily notes → standalone files
9. **/closeday** — Capture what happened today, progress, new ideas, carry-overs
10. **/drift** — Surface loosely connected recurring themes across unrelated notes
11. **/emerge** — Identify pattern clusters becoming projects/essays/products
12. **/schedule** — Map priorities to time blocks, flag conflicts

## Key Philosophy
- Quality of context determines agent capability
- Vault is the context — interlinked markdown files
- Link graph relationships are first-class (backlinks, connections)
- Commands are thinking tools, not just queries
- Start with daily writing; commands work better with more context
